# Execution Plan: Website & Bayesian Dashboard Overhaul

## Objective
Analyze Dennis J. Carroll's personal website and Bayesian analysis dashboard, then produce a vivid, detailed execution plan for improvements that:
1. Showcases data science capability to recruiters
2. Makes the Bayesian dashboard appealing to a general audience
3. Enables CSV upload and broader usability
4. Integrates the project into the homepage with interactive mini-previews

## Stage 1 — Parallel Deep Analysis (3 sub-agents)

### Agent 1: Personal Website Analyst
- Analyze homepage UX, visual design, project showcase approach
- Identify gaps in how the Bayesian project is represented
- Benchmark against top data science portfolio sites
- Assess the "Featured Apps" section and project preview patterns

### Agent 2: Bayesian Dashboard Analyst  
- Analyze the Streamlit dashboard's 6 pages, UX, and visual design
- Identify barriers to general audience engagement
- Assess CSV upload feasibility and architecture changes needed
- Evaluate the "sleep" problem and hosting alternatives

### Agent 3: GitHub Codebase Analyst
- Analyze the repo structure, model architecture, and dashboard code
- Identify the coupling to Seattle Airbnb data
- Assess what it would take to generalize for any CSV
- Review the dashboard theming and visual polish

## Stage 2 — Synthesis & Execution Plan Writing
- Combine all analyses into a vivid, detailed execution outline
- Prioritize improvements by impact and effort
- Include specific technical recommendations, design direction, and feature specs
- Output: Comprehensive markdown report with visual design direction

## Skills Used
- `deep-research-swarm` (Stage 1 — for benchmarking and best practices)
- `report-writing` (Stage 2 — for the execution plan document)
- `docx` (Stage 3 — final document conversion if needed)
